# Codex Working Rules

## Default Rule

- 除非用户明确说明“先做测试版 / 不发布正式版”，否则所有改动完成后默认一起发布到公开正式版。

## Problem-Solving Rule

- 默认使用第一性原理处理问题。
- 先定位根因，再决定最短修复路径，不堆叠无效步骤。
- 优先选择能直接消除根因、同时减少后续维护成本的方案。
- 修改后必须做快速验证，验证失败就继续收敛，不把未确认结果当成完成。

## Execution Rule

- 默认同时完成：修复、回归检查、必要的数据/缓存更新、正式发布。
- 只有在高风险、缺权限、或用户明确要求保留测试态时，才中断在“只改代码不发布”的阶段。
